# Fairtale 디자인 원칙

## 목적

이 문서는 Fairtale의 랜딩페이지, 브랜드 자료, MVP 화면을 만들 때 지켜야 할 디자인 방향을 정리한다.

Fairtale은 단순한 AI 통역 툴이 아니라, 해외진출이 막막한 기업이 실제 해외 바이어와 만날 수 있도록 돕는 B2B 서비스다. 따라서 디자인은 기술적으로 멋져 보이는 것보다, 사용자가 "여기라면 한번 맡겨봐도 되겠다"는 신뢰와 쉬움을 느끼게 해야 한다.

## 핵심 디자인 방향

Fairtale의 디자인은 다음 세 가지 감각을 동시에 가져야 한다.

1. 신뢰감 있는 B2B 서비스
2. 해외진출이 쉬워지는 안내자
3. 미래지향적인 AI 지원 경험

하지만 이 중 가장 중요한 것은 신뢰와 쉬움이다. AI와 글래스모피즘 스타일은 보조적인 표현이어야 한다.

## 랜딩페이지 기본 원칙

랜딩페이지는 플랫폼 설명 페이지가 아니라, 공급 기업의 신청을 받는 전환 페이지에 가깝다.

방문자는 첫 화면에서 바로 다음을 이해해야 한다.

> "제품을 등록하면 Fairtale이 해외 바이어 상담 준비를 도와준다."

따라서 랜딩페이지는 다음 원칙을 따른다.

- 메인 메시지는 수출기업 중심으로 명확하게 유지한다.
- 바이어용 안내는 별도 페이지로 유도한다.
- 히어로 섹션은 깨끗하고 밝게 유지한다.
- CTA는 "가입하기"보다 "무료 제품 검토 요청"처럼 결과 중심으로 쓴다.
- 사용자가 받을 결과물을 시각적으로 보여준다.
- 기술 설명보다 사용자에게 생기는 이득을 먼저 보여준다.

## 메인 랜딩 구조

추천 구조:

1. Header
   - 로고
   - 수출기업
   - 진행 방식
   - 파트너
   - 바이어용
   - 문의

   - 로그인
   - CTA: 무료 제품 검토 요청

2. Hero

   - 제품 검토 요청 CTA
   - 샘플 결과물 보기
   - 오른쪽에는 Fairtale이 만들어주는 결과물 미리보기 패널

3. Buyer Route Banner
   - 바이어를 위한 별도 페이지 유도
   - 메인 전환을 방해하지 않는 얇은 배너 형태

4. 등록 후 받을 수 있는 것
   - 영문 제품 소개

   - 추천 바이어 유형
   - 상담 준비 체크리스트

5. 진행 방식
   - 제품 등록
   - Fairtale 검토
   - 상담 준비
   - 미팅 연결
   - 팔로업

## 히어로 카피 원칙

히어로는 수출기업에게 직접 말해야 한다.

추천 메인 카피:

> 제품만 등록하세요. 해외 바이어 상담 준비는 Fairtale이 도와드립니다.

추천 서브 카피:

> 영문 제품 소개, 해외 판매 포인트, 추천 바이어 유형까지 Fairtale이 정리해드립니다.

CTA 후보:

- 무료로 제품 검토 요청하기
- 제품 등록하고 바이어 상담 준비하기
- 내 제품의 해외 바이어 유형 확인하기

상단 CTA는 짧게 쓰고, 히어로 CTA는 조금 더 설명적으로 쓴다.

예:

- Header CTA: 제품 검토 요청
- Hero CTA: 무료로 제품 검토 요청하기

## 바이어 유도 배너

메인 랜딩은 수출기업 중심으로 두되, 바이어가 들어왔을 때 이탈하지 않도록 별도 진입점을 제공한다.

추천 문구:

> 아시아 공급사를 찾고 계신가요?

> 관심 카테고리를 알려주시면 Fairtale이 적합한 공급사 후보를 큐레이션해드립니다.

CTA:

> 바이어로 시작하기

이 배너는 히어로와 두 번째 섹션 사이에 둔다. 시각적으로 구분되지만 메인 CTA보다 강하면 안 된다.

## 글래스모피즘 적용 원칙

RonDesignLab 스타일의 글래스모피즘은 Fairtale에 잘 맞을 수 있지만, 전체 페이지에 과하게 적용하면 안 된다.

적용하면 좋은 곳:

- 히어로 오른쪽의 제품 검토 리포트 패널
- Fairtale이 만들어주는 결과물 카드
- 바이어 추천 카드
- 진행 상태 UI
- 관리자/내부 운영 화면

피해야 할 곳:

- 히어로 전체 배경
- 모든 섹션 카드
- CTA 버튼
- 긴 텍스트 본문
- 핵심 메시지 주변

원칙:

> 페이지 전체는 쉽고 신뢰감 있게, 글래스모피즘은 "Fairtale이 만들어주는 결과물"에만 적용한다.

## 글래스모피즘 스타일 가이드

사용 권장:

- 밝은 배경 위의 반투명 흰색 카드
- 1px의 옅은 border
- 매우 부드러운 shadow
- 약한 blur 느낌
- 충분한 여백
- 작은 mint/teal 포인트

구현 기준:

- 카드 배경: `rgba(255, 255, 255, 0.45)`에서 `rgba(255, 255, 255, 0.70)` 사이
- blur: `backdrop-filter: blur(10px)` 이상. 핵심 리포트 패널은 16-24px까지 사용 가능
- border: `1px solid rgba(255, 255, 255, 0.45)`와 옅은 navy/mint 계열 border를 함께 고려
- shadow: 짧고 진한 그림자보다 넓고 부드러운 그림자를 사용
- 내부 카드: 완전한 흰색보다 `white/55`, `white/65`처럼 반투명 계층을 유지
- 배경: blur가 보이려면 카드 뒤에 흐릿하게 보일 색상/라인/정보형 레이어가 필요하다

피해야 할 것:

- 어두운 풀페이지 배경
- 과한 그라데이션
- 보라색/퍼플 계열 SaaS 느낌
- 장식용 블롭, 오브, 빛 효과
- 너무 많은 floating card
- 가독성을 해치는 투명도

글래스모피즘이 약해 보이는 대표적인 원인:

- 카드 뒤가 단색 배경이라 blur가 눈에 띄지 않음
- 모든 내부 카드가 완전한 흰색이라 유리의 계층감이 사라짐
- border가 밝은 배경과 섞여 유리판의 경계가 보이지 않음
- shadow가 너무 약하거나 너무 짧아 떠 있는 느낌이 부족함
- 글래스 효과를 적용했지만 주변에 굴절될 정보형 배경이 없음

제품 검토 리포트 패널 개선 방향:

1. 패널 뒤에 아주 은은한 mint/teal/navy 색상 레이어 또는 얇은 정보형 라인을 배치한다.
2. 외부 패널은 `white/50~60` 정도의 반투명 배경과 강한 blur를 사용한다.
3. 내부 카드들은 `white/55~70` 정도로 낮추고, border와 shadow로 계층을 만든다.
4. 배경 장식은 블롭/오브처럼 보이면 안 되고, 리포트 UI의 일부처럼 느껴져야 한다.
5. 텍스트가 올라가는 영역은 투명도를 너무 낮추지 않아 가독성을 유지한다.

## 컬러 원칙

기본 팔레트:

- Deep Navy: `#153156`
- Fairtale Teal: `#12978B`
- Soft Mint: `#E6F8F4`
- Off White: `#F7F9FC`
- White: `#FFFFFF`
- Light Border: `#DFE5EE`

사용 방식:

- 로고와 주요 텍스트는 Deep Navy를 사용한다.
- CTA는 Teal 또는 Deep Navy를 사용한다.
- 배경은 Off White 또는 White를 기본으로 한다.
- Soft Mint는 바이어 배너, 리포트 카드 배경, 보조 강조에 사용한다.
- Warm Gold는 현재 로고에서 제외했으므로, 아주 특별한 성과/기회 포인트가 아니라면 남발하지 않는다.

## 타이포그래피 원칙

Fairtale의 기본 폰트는 Pretendard로 한다.

사용 원칙:

- 웹, 랜딩페이지, 앱 UI, 발표 자료의 기본 한글/영문 폰트는 `Pretendard`를 우선 사용한다.
- 시스템 fallback은 `Inter`, `-apple-system`, `BlinkMacSystemFont`, `Segoe UI`, `sans-serif` 순서로 둔다.
- 히어로 헤드라인과 큰 섹션 제목은 브랜드성 강화를 위해 `GyeongbokgungSumunjangTitle`을 사용한다.
- 제목용 폰트 fallback은 `Chiron Sung HK`, `Pretendard`, `Noto Sans KR`, `sans-serif` 순서로 둔다.
- 히어로 헤드라인은 크고 명확하게 사용하되, 글자 크기보다 줄 간격과 가독성을 우선한다.
- 본문은 너무 얇은 웨이트를 피하고, 최소 `Regular` 이상을 사용한다.
- CTA와 메뉴는 `SemiBold` 또는 `Bold`를 사용해 클릭 가능한 요소처럼 보이게 한다.
- 카드 제목, 리포트 내부 라벨, 프로세스 단계명은 대제목이 아니므로 Pretendard를 사용한다.

현재 구현 기준:

- 기본 폰트: `Pretendard`
- 대제목 폰트: `GyeongbokgungSumunjangTitle`
- 대제목 적용 범위:
  - 히어로 메인 카피
  - `등록 후 받을 수 있는 것`
  - `어렵지 않게, 단계별로 진행합니다`
  - `바이어를 만나는 것에서 끝나지 않습니다`
  - `무료 제품 검토를 요청하세요`
- 본문 폰트 유지 범위:
  - `제품 검토 리포트`
  - `영문 소개`
  - `판매 포인트`
  - `추천 바이어 유형`
  - `상담 준비 체크리스트`
  - `제품 정보 등록`
  - `Fairtale 팀 검토`
  - `바이어용 소개 자료 정리`
  - `상담 가능성 확인`

CSS 예시:

```css
font-family: Pretendard, Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

제목 웹폰트는 브라우저 호환성을 위해 공식 안내된 `woff2` 파일을 사용한다. 로컬 `otf/ttf` 파일은 Chrome OTS 검증에서 `kern: Bad subtable` 오류로 거부될 수 있으므로 랜딩 구현에는 사용하지 않는다.

## 로고 사용 원칙

현재 로고 방향:

- Gateway / Open Door 심볼
- 작은 별 1개
- 딥 네이비 단색
- 워드마크: `Fairtale`

의미:

- 해외진출의 첫 관문
- 새로운 시장으로 나아가는 길
- 작지만 분명한 기회

사용 원칙:

- 로고는 밝은 배경 위에서 사용한다.
- 로고 색상은 기본적으로 Deep Navy `#153156`을 사용한다.
- 로고 주변 여백을 충분히 둔다.
- 별 포인트를 별도 장식 요소처럼 반복 사용하지 않는다.
- 로고를 과한 그라데이션이나 그림자와 함께 쓰지 않는다.

## 비주얼 콘텐츠 원칙

랜딩페이지에서 제품 이미지를 사용할 때는 특정 산업으로 오해되지 않도록 주의한다.

예를 들어 화장품 제품 이미지만 크게 쓰면 Fairtale이 뷰티 수출 플랫폼처럼 보일 수 있다. 초기에는 특정 제품 사진보다, "제품 카드"나 "검토 리포트" 형태의 추상화된 UI가 더 안전하다.

사용하면 좋은 비주얼:

- 제품 검토 리포트 UI
- 바이어 추천 카드
- 상담 준비 체크리스트
- 영문 제품 소개 샘플
- 진행 단계 UI

조심해야 할 비주얼:

- 특정 업종 제품 사진
- 글로벌 느낌을 위한 지구본
- 비행기, 악수, 컨테이너 같은 흔한 무역 아이콘
- 과하게 미래적인 AI 대시보드 이미지

## 메뉴 구조 원칙

메인 랜딩의 상단 메뉴는 공급 기업 중심으로 구성하되, 바이어용 진입점은 분명히 열어둔다.

추천 메뉴:

- 수출기업
- 진행 방식
- 파트너
- 바이어용
- 문의
- 언어 변경
- 로그인

CTA:

- 제품 검토 요청

상단 오른쪽 구성은 다음처럼 둔다.

- `KO ▾`: 언어 변경 진입점. 메인 CTA보다 약하게 표시한다.
- `로그인`: 기존 고객용 진입점. CTA처럼 강하게 보이지 않게 텍스트 링크나 옅은 버튼으로 처리한다.
- `제품 검토 요청`: 신규 수출기업 전환을 위한 주요 CTA. 가장 눈에 띄는 버튼으로 처리한다.

초기 MVP에 실제 로그인/대시보드 기능이 없다면 랜딩에서 로그인은 숨겨도 된다. 로그인 기능이 준비되는 순간 상단 우측에 추가하되, 신규 신청 CTA보다 시각적 우선순위는 낮게 둔다.

바이어용 설명은 메인 랜딩 안에서 길게 다루지 않고 별도 페이지로 분리한다.

## 언어 변경 원칙

Fairtale은 한국/아시아 수출기업과 해외 바이어를 함께 다루는 서비스이므로 언어 변경 기능이 필요하다.

다만 메인 랜딩의 주요 행동은 `제품 검토 요청`이므로, 언어 변경 버튼은 CTA보다 약하게 보여야 한다.

추천 Header 우측 구성:

- `KO ▾`
- `로그인`
- `제품 검토 요청`

초기 언어 옵션:

- KO
- EN

확장 후보:

- DE
- JP
- ZH

초기 우선순위:

1. 한국어 메인 랜딩
2. 영어 바이어용 페이지
3. 영어 전체 랜딩
4. 독일어/일본어/중국어 확장

언어 버튼은 작고 명확하게 둔다. 별도의 강한 버튼처럼 만들지 않고, 텍스트 링크나 가벼운 셀렉트 형태로 처리한다.

## 디자인 판단 기준

새로운 화면이나 섹션을 만들 때 다음 질문으로 판단한다.

1. 수출기업이 "내가 무엇을 하면 되는지" 바로 이해하는가?
2. Fairtale이 제공하는 결과물이 구체적으로 보이는가?
3. 신뢰감과 쉬움이 기술적 화려함보다 먼저 느껴지는가?
4. 바이어용 진입점이 있지만 메인 전환을 방해하지 않는가?
5. 글래스모피즘이 정보 전달을 돕고 있는가, 아니면 장식이 되고 있는가?
6. 특정 산업 서비스처럼 오해될 만한 비주얼은 없는가?

## 현재 결론

Fairtale의 랜딩페이지는 다음 방향이 가장 적합하다.

> 메인 랜딩은 수출기업을 위한 신청 전환 페이지로 설계하고, 바이어는 별도 페이지로 유도한다. 전체 톤은 밝고 신뢰감 있게 유지하며, 글래스모피즘은 Fairtale이 제공하는 결과물 미리보기와 리포트 카드에만 절제해서 사용한다.

## 참고 시안

현재 랜딩페이지 디자인 방향의 기준 시안:

이 시안은 다음 원칙을 보여주는 참고 이미지다.
