- **MVP 목표**: 1:1 실시간 음성통역 화상회의 시스템

## 📊 **현재 진행 상황 (2026-04-27 코드 점검 기준)**
- **전체 진행률**: 약 40-50% 완료
- **현재 상태**: 1인 로컬 음성 번역 PoC 수준. 2인 실시간 화상통역 MVP로 보기에는 핵심 연결부가 아직 미완성.
- **완료에 가까운 부분**: PIN 기반 룸 입장, 최대 2인 룸 제한, 로컬 미디어 스트림, 오디오 캡처, STT/번역/TTS 모듈 코드, 자막 UI
- **가장 큰 미완성 부분**: 서버 WebRTC signaling 중계, 양방향 언어 처리, 프로덕션 빌드 통과, 2인 E2E 검증
- **다음 우선순위**: WebRTC signaling 서버 구현 → Next.js build 실패 수정 → 2인 화상 연결 검증 → 양방향 통역 처리

## 🚨 **현재 MVP 현실 점검**
- **MVP 판단**: "실시간 통역 화상회의 MVP"보다는 "1인 음성 번역 기술검증 PoC"에 가까움.
- **2인 화상 연결 리스크**: 클라이언트는 `offer`, `answer`, `ice-candidate`를 송수신하도록 구현되어 있으나, 서버에는 해당 Socket.IO 이벤트 핸들러가 없음.
- **양방향 통역 리스크**: 클라이언트 오디오 전송 언어가 `ko-KR`로 고정되어 있고, 서버 TTS도 `EN`으로 고정되어 있음.
- **배포 리스크**: `npm run build` 실행 시 `client/src/hooks/useSocketTester.ts`의 `socket.once` 타입 오류로 실패함.
- **테스트 리스크**: 1인 테스트 기록은 있으나, 2개 브라우저/2개 디바이스에서 실제 WebRTC + STT + 번역 + TTS가 함께 동작한 검증은 아직 없음.

## 🔥 **MVP 완성 전 필수 수정사항**
1. 서버 `index.js`에 WebRTC signaling 이벤트 추가: `offer`, `answer`, `ice-candidate`
2. `client/src/hooks/useSocketTester.ts` 타입 오류 수정 또는 테스트 훅 정리
3. Socket 서버 주소를 `localhost:4000` 고정값에서 환경변수 기반으로 변경
4. 사용자별 언어 설정 구조화: 내 언어, 상대 언어, STT 언어, 번역 대상 언어, TTS 언어
5. TTS/자막 전송 대상을 테스트용 전체 전송에서 실제 상대방 중심 전송으로 변경
6. 2인 브라우저 탭 테스트와 노트북+모바일 테스트로 E2E 검증
7. Vercel/Railway 배포 환경에서 HTTPS, CORS, WebRTC 연결 확인
